# Формулы и правила расчётов модуля «Деньги / ДДС»

Файл описывает расчёты, используемые в `money.html` и `gsmoney.gs`.

## 1. Источник данных

- Основной лист: `Деньги`.
- Дата операции извлекается из поля `Регистратор`.
- Сумма операции берётся из поля `Результ`.
- Основные аналитики:
  - `Направление`
  - `ЦФО`
  - `Статья ДДС.Родитель`
  - `Статья ДДС`
  - `Верхний уровень`

## 2. Исключения из расчётов

Из оборотов исключаются внутренние перемещения:

- `Перемещение денег`
- `Движение денег внутри ШВО`

Проверка выполняется по тексту:

- `Статья ДДС`
- `Статья ДДС.Родитель`
- `Верхний уровень`

### Контроль внутренних перемещений

Внутренние перемещения не участвуют в ОДДС и не раздувают обороты, но их нетто-сумма контролируется отдельно.

```text
Нетто_Внутренних_Перемещений =
  SUM(Результ по статьям "Перемещение денег" и "Движение денег внутри ШВО")
```

Если:

```text
Нетто_Внутренних_Перемещений != 0
```

то показывается warning:

```text
Внутренние перемещения не сошлись
```

Это означает, что в базе может быть неполная пара перемещений или ошибка разнесения.

## 3. Учебный календарь

Учебный год считается с сентября по август.

### Учебный год

Если месяц операции `>= 9`:

```text
Учебный_Год = Год + "-" + (Год + 1)
```

Если месяц операции `<= 8`:

```text
Учебный_Год = (Год - 1) + "-" + Год
```

### Номер месяца учебного года

```text
Если Месяц >= 9:
  Номер_Месяца_Учебного = Месяц - 8

Если Месяц <= 8:
  Номер_Месяца_Учебного = Месяц + 4
```

Пример:

- Сентябрь = `1`
- Октябрь = `2`
- Август = `12`

### Период месяца

Любая дата операции приводится к первому числу месяца:

```text
Период_Месяца = первое число месяца операции
```

## 4. Направление и ЦФО

Если `Направление` пустое:

```text
Направление = "Администрация / Общее"
```

Если `ЦФО` пустое:

```text
ЦФО = "Не распределено"
```

Фиксированные направления в фильтре:

- `Азарово`
- `Северная`
- `Валуйки`
- `Администрация / Общее`

## 5. Разделы ОДДС

Каждая операция распределяется в один из трёх разделов.

### Важное правило приоритета

Сначала учитывается поле `Верхний уровень`:

- если `Верхний уровень = "Выручка"` — строка относится к `Поступления`;
- если `Верхний уровень = "Затраты/расходы"` — строка относится к `Платежи`.
- если `Верхний уровень = "Кредиты"`:
  - `Результ >= 0` — финансовая деятельность / поступления;
  - `Результ < 0` — финансовая деятельность / платежи.
- если `Верхний уровень = "Выяснить"`:
  - `Результ >= 0` — текущая деятельность / поступления / `Прочие доходы`;
  - `Результ < 0` — текущая деятельность / платежи / `Выяснить`;
  - суммы отдельно выводятся маркером `Требуется разбор статьи «Выяснить»`;
  - если сумма равна нулю, маркер не отображается.
- если `Верхний уровень = "Подотчетные суммы"`:
  - `Результ >= 0` — текущая деятельность / поступления / `Возврат подотчетных сумм`;
  - `Результ < 0` — текущая деятельность / платежи / `Подотчетные суммы`.

Текстовые правила по названию статьи применяются только как дополнительная классификация, если по `Верхний уровень` нельзя уверенно определить тип потока.

### 5.1 Текущая деятельность

Поступления:

- `Основное образование`
- `Доп. Образование`
- `Дополнительное образование`
- `Дополнительные доходы/расходы`
- `Проекты`
- `Лагерь`
- `Мероприятия`
- `Прочие доходы`
- `Субсидия`
- `Депозиты`

Платежи:

- `Аренда`
- `Прочие расходы`
- `Реклама`
- `Поставщикам`
- `Материалы`
- `Коммуналка`
- `Электроэнергия`
- `Налоги`
- `ФОТ`
- `Проценты по кредиту`
- `Выдача в подотчет`

### 5.2 Инвестиционная деятельность

Платежи:

- `Приобретение ОС`
- `Основные средства`
- `Лизинг`

### 5.3 Финансовая деятельность

Поступления:

- `Взнос в УК`
- `Получение ссуды`
- `Получение кредита`

Платежи:

- `Погашение ссуды`
- `Погашение кредита`
- `Дивиденды`

## 6. Денежные потоки

### Поступления

```text
Поступления = Результ
если Верхний уровень = "Выручка"
```

Если по доходной статье есть минусовая корректировка или возврат, она уменьшает поступления этой же статьи.

### Выплаты

```text
Выплаты = -Результ
если Верхний уровень = "Затраты/расходы"
```

Если по расходной статье есть плюсовая корректировка или возврат, она уменьшает выплаты этой же статьи.

### Значение ОДДС

```text
Если тип потока = "Поступления":
  Значение_ОДДС = Результ

Если тип потока = "Платежи":
  Значение_ОДДС = Результ
```

Иными словами, для строк ОДДС сохраняется знак исходного поля `Результ`.

### ЧДП раздела ОДДС

```text
ЧДП_Раздела = SUM(Значение_ОДДС по разделу)
```

### Общий ЧДП периода

```text
Общий_ЧДП = ЧДП_Текущая + ЧДП_Инвестиционная + ЧДП_Финансовая
```

В коде карточка общего ЧДП считается как:

```text
Общий_ЧДП = SUM(Значение_ОДДС по текущему срезу фильтров)
```

## 7. Остатки денежных средств

Стартовый баланс:

```text
START_BALANCE = 4 311 060,98
Дата старта = 01.09.2025
```

### Остаток на начало месяца

```text
Остаток_На_Начало_Месяца =
  START_BALANCE
  + SUM(Общий ЧДП периода / Значение_ОДДС с 01.09.2025 до начала месяца)
```

Из суммы исключены внутренние перемещения денег.

### Оборот месяца

```text
Оборот_Месяца = SUM(Общий ЧДП периода / Значение_ОДДС за месяц)
```

### Остаток на конец месяца

```text
Остаток_На_Конец_Месяца =
  Остаток_На_Начало_Месяца + Оборот_Месяца
```

### Валидация баланса

```text
Остаток_На_Начало_Следующего_Месяца
=
Остаток_На_Начало_Текущего_Месяца + Оборот_Текущего_Месяца
```

Расхождение допускается только в пределах копейки.

## 8. Cash Buffer

Формула:

```text
Cash_Buffer =
  Остаток денежных средств на начало текущего месяца
  /
  Среднедневной расход за предыдущие 30 дней
```

Расход для среднего считается только по платежам текущей деятельности:

```text
Среднедневной_Расход =
  SUM(-Значение_ОДДС платежей текущей деятельности за предыдущие 30 дней)
  / 30
```

Плюсовые возвраты по расходным статьям уменьшают расходную базу.

Если расходов нет:

```text
Cash_Buffer = 0 / "—"
```

## 9. Коэффициент авансирования

Назначение: защита от ложной ликвидности.

### База поступлений

```text
База_Поступлений =
  SUM(Поступления)
  где Статья ДДС.Родитель содержит "Основное образование"
```

### База обязательств

```text
База_Обязательств =
  SUM(Выплаты)
  где Статья ДДС.Родитель или Статья ДДС содержит "ФОТ" или "Аренда"
```

### Формула

```text
Коэффициент_Авансирования =
  База_Поступлений / База_Обязательств
```

Если база обязательств равна нулю:

```text
Коэффициент_Авансирования = "—"
```

Интерпретация:

- `> 1.0` — авансы превышают жёсткие обязательства, есть риск тратить деньги будущих периодов.
- `< 1.0` — авансовая подушка снижена или исчерпана.

## 10. Направленная маржинальность денежного потока

Назначение: оценка эффективности направления или всей группы компаний.

### ЧДП направления

```text
NCF_Направления =
  SUM(Поступления направления)
  -
  SUM(Выплаты направления)
```

### Сумма поступлений направления

```text
Поступления_Направления =
  SUM(Поступления)
  где Верхний уровень = "Выручка"
```

### Формула

```text
Маржинальность_ДП =
  NCF_Направления / Поступления_Направления * 100%
```

Если поступления равны нулю:

```text
Маржинальность_ДП = "—"
```

Карточка пересчитывается по текущим фильтрам страницы.

## 11. Диаграмма «Зависимость от ЦФО»

Режимы:

- `Поступления`
- `Выплаты`

### Сортировка

```text
ЦФО сортируются по убыванию абсолютной суммы
```

### Маркер внимания 30%

```text
Доля_ЦФО = ABS(Сумма_ЦФО) / SUM(ABS(Сумма всех ЦФО))
```

Если:

```text
Доля_ЦФО > 30%
```

то столбец подсвечивается красным.

## 12. Коэффициент административной нагрузки

Показывается во всплывающей подсказке для административных ЦФО.

Административные ЦФО определяются по словам:

- `АХО`
- `Финансов`
- `HR`
- `Администрац`
- `Руковод`
- `Бухгалтер`
- `Кадры`
- `Маркетинг`
- `Офис`

Учебные ЦФО определяются по словам:

- `Ступень`
- `Класс`
- `Группа`
- `Гр`

Формула:

```text
Коэффициент_Административной_Нагрузки =
  Расход текущего административного ЦФО
  /
  SUM(Расходов всех учебных ступеней, классов и групп)
  * 100%
```

Если база расходов учебных ступеней равна нулю:

```text
Коэффициент = "нет базы расходов по учебным ступеням"
```

## 13. Контроль ЦФО «Не распределено»

Считается только по выплатам.

```text
Доля_Не_Распределено =
  Выплаты по ЦФО "Не распределено"
  /
  Все выплаты
  * 100%
```

Если:

```text
Доля_Не_Распределено > 5%
```

то показывается красный warning-флаг.

## 14. ЧДП по направлениям

Для каждого направления:

```text
ЧДП_Направления =
  SUM(Значение_ОДДС по направлению)
```

Дополнительно считается маржинальность:

```text
Маржинальность_Направления =
  (Поступления_Направления - Выплаты_Направления)
  /
  Поступления_Направления
  * 100%
```

Если поступлений нет:

```text
Маржинальность_Направления = "—"
```

## 15. Структура расходов

Используются только строки:

```text
Верхний уровень = "Затраты/расходы"
```

Режимы группировки:

- `Категории` — по `Статья ДДС.Родитель`
- `Направления` — по `Направление`
- `ЦФО` — по `ЦФО`

Формула процента:

```text
Доля_Категории =
  Выплаты категории / SUM(Выплат всех категорий) * 100%
```

## 16. Матрица ОДДС

Структура строк:

1. `Остаток на начало периода`
2. `Раздел 1. Денежные потоки от текущей деятельности`
   - `Поступления`
   - `Платежи`
   - `ЧДП от текущей деятельности`
3. `Раздел 2. Денежные потоки от инвестиционной деятельности`
   - `Поступления`
   - `Платежи`
   - `ЧДП от инвест. деятельности`
4. `Раздел 3. Денежные потоки от финансовой деятельности`
   - `Поступления`
   - `Платежи`
   - `ЧДП от фин. деятельности`
5. `Общий ЧДП периода`
6. `Остаток на конец периода`

Столбцы:

- месяцы учебного года;
- сортировка по `Номер_Месяца_Учебного`.

## 17. Drill-down

При двойном клике по ячейке матрицы открывается детализация.

В детализацию передаются текущие фильтры:

- период;
- направление;
- ЦФО;
- родительская статья;
- статья;
- месяц;
- раздел ОДДС;
- тип потока.

Показываются поля:

- `Дата`
- `Регистратор`
- `Результат`
- `ЦФО`
- `Комментарий`

## 18. Защита от ошибок

Все деления выполняются через безопасную проверку:

```text
Если знаменатель = 0:
  вернуть "—"
```

Это защищает интерфейс от:

- `NaN`
- `Infinity`
- ошибок отображения при пустом периоде или направлении без оборотов.

## 18.1 Сверка листа «Деньги» с итогом ОДДС

Дашборд проверяет, что нетто-сумма источника совпадает с итогом ОДДС.

```text
Нетто_Листа_Деньги =
  SUM(Результ по всем строкам, кроме внутренних перемещений)
```

```text
Нетто_ОДДС =
  SUM(Значение_ОДДС)
```

```text
Расхождение =
  Нетто_Листа_Деньги - Нетто_ОДДС
```

Если:

```text
Расхождение != 0
```

то показывается warning:

```text
Итог листа «Деньги» не совпал с ОДДС
```

Это означает, что часть верхнеуровневых статей не классифицирована или попала не в тот раздел.

## 19. Как сверять данные с исходным ОДДС

Для сверки добавлена Apps Script-функция:

```text
cashCreateMoneyReconciliationReport()
```

Она создаёт отдельный лист:

```text
Проверка ДДС
```

В листе строки группируются по:

- месяц;
- учебный год;
- направление;
- ЦФО;
- `Статья ДДС.Родитель`;
- `Статья ДДС`;
- `Верхний уровень`;
- раздел ОДДС;
- тип потока.

Для каждой группы считаются:

- количество исходных строк;
- сумма `Результ`;
- сумма `Поступления`;
- сумма `Выплаты`.

### Рекомендуемый порядок проверки расхождения

1. Запустить функцию `cashCreateMoneyReconciliationReport()` в редакторе Apps Script.
2. Открыть лист `Проверка ДДС`.
3. Поставить фильтры:
   - месяц = нужный месяц;
   - ЦФО = нужный ЦФО;
   - `Статья ДДС.Родитель` = нужная родительская статья;
   - при необходимости `Статья ДДС` = нужная детальная статья.
4. Сравнить сумму `Поступления` или `Выплаты` с дашбордом.

### Важное правило

Если в дашборде выбрана диаграмма `Зависимость от ЦФО`, она показывает сумму по ЦФО в рамках режима:

```text
Поступления = все строки Верхний уровень = "Выручка"
Выплаты = все строки Верхний уровень = "Затраты/расходы"
```

То есть значение по ЦФО может быть больше, чем сумма только по одной родительской статье, например `Основное образование`.
