Формулы модуля «Деньги / ДДС»

Печатная инструкция для проверки расчетов дашборда. Источник: лист «Деньги» в реестре ДДС.

1. Базовые правила

Источник

  • Лист: Деньги
  • Дата: извлекается из поля Регистратор
  • Сумма: поле Результ

Исключения

  • Перемещение денег
  • Движение денег внутри ШВО
Внутренние перемещения исключаются из ОДДС, но контролируются отдельно: если их нетто-сумма не равна нулю, дашборд показывает warning «Внутренние перемещения не сошлись».

2. Учебный календарь

Если месяц >= 9: Учебный_Год = Год + "-" + (Год + 1) Если месяц <= 8: Учебный_Год = (Год - 1) + "-" + Год
Если месяц >= 9: Номер_Месяца_Учебного = Месяц - 8 Если месяц <= 8: Номер_Месяца_Учебного = Месяц + 4

3. ОДДС: разделы и типы потоков

Приоритет классификации: если «Верхний уровень» = «Выручка», строка относится к поступлениям; если «Верхний уровень» = «Затраты/расходы», строка относится к платежам; если «Верхний уровень» = «Кредиты», положительный результат относится к финансовым поступлениям, отрицательный — к финансовым платежам; если «Верхний уровень» = «Выяснить», положительный результат временно относится к текущей деятельности / поступлениям / «Прочие доходы», отрицательный — к текущей деятельности / платежам / «Выяснить»; если «Верхний уровень» = «Подотчетные суммы», положительный результат относится к возврату подотчетных сумм, отрицательный — к платежам подотчета. Текстовые правила по статьям применяются только как дополнительная классификация.
Раздел Поступления Платежи
Текущая деятельность Основное образование, Доп. образование, проекты, лагерь, мероприятия, прочие доходы, субсидия, депозиты Аренда, прочие расходы, реклама, поставщики, коммуналка, налоги, ФОТ, проценты по кредиту, подотчет
Инвестиционная деятельность — Приобретение ОС, основные средства, лизинг
Финансовая деятельность Взнос в УК, получение ссуды, получение кредита Погашение ссуды, погашение кредита, дивиденды

4. Основные денежные формулы

Поступления = Результ если Верхний уровень = "Выручка"
Выплаты = -Результ если Верхний уровень = "Затраты/расходы"
Если тип потока = "Поступления": Значение_ОДДС = Результ Если тип потока = "Платежи": Значение_ОДДС = Результ
Для строк ОДДС сохраняется знак исходного поля «Результ»: минусовые корректировки доходов уменьшают поступления, плюсовые возвраты расходов уменьшают выплаты.
Общий_ЧДП = SUM(Значение_ОДДС по текущему срезу фильтров)

5. Остатки денежных средств

START_BALANCE = 4 311 060,98 на дату 01.09.2025.

Остаток_На_Начало_Месяца = START_BALANCE + SUM(Общий ЧДП периода / Значение_ОДДС с 01.09.2025 до начала месяца)
Оборот_Месяца = SUM(Общий ЧДП периода / Значение_ОДДС за месяц)
Остаток_На_Конец_Месяца = Остаток_На_Начало_Месяца + Оборот_Месяца
Контроль: остаток на начало следующего месяца должен равняться остатку на начало текущего месяца плюс оборот текущего месяца.

6. KPI верхнего уровня

KPI Формула Защита
Cash Buffer Остаток на начало текущего месяца / среднедневной расход за предыдущие 30 дней Если расходов нет — выводится «—»
Коэффициент авансирования Поступления по «Основное образование» / выплаты по «ФОТ + Аренда» Если база обязательств = 0 — «—»
Маржинальность ДП (Поступления - Выплаты) / Поступления × 100% Если поступлений нет — «—»

7. Диаграмма ЦФО

Доля_ЦФО = ABS(Сумма_ЦФО) / SUM(ABS(Сумма всех ЦФО))

Если доля ЦФО больше 30%, столбец подсвечивается красным.

Коэффициент_Административной_Нагрузки = Расход текущего административного ЦФО / SUM(Расходов всех учебных ступеней, классов и групп) × 100%
Доля_Не_Распределено = Выплаты по ЦФО "Не распределено" / Все выплаты × 100%

Если доля «Не распределено» больше 5%, показывается красный warning-флаг.

8. ЧДП по направлениям

ЧДП_Направления = SUM(Значение_ОДДС по направлению)
Маржинальность_Направления = (Поступления_Направления - Выплаты_Направления) / Поступления_Направления × 100%

9. Структура расходов

Используются только строки, где Верхний уровень = «Затраты/расходы».

Доля = Выплаты выбранной группы / SUM(Выплат всех групп) × 100%

Режимы группировки: категории, направления, ЦФО.

10. Drill-down и безопасность

  • Детализация открывается по двойному клику на ячейку матрицы.
  • В детализацию передаются текущие фильтры страницы.
  • Показываются: дата, регистратор, результат, ЦФО, комментарий.
  • Все деления защищены: если знаменатель равен нулю, выводится «—».
  • Итог листа «Деньги» сверяется с итогом ОДДС. Если нетто-суммы не совпали, дашборд показывает warning.
